---
title: API Client
description: Type-safe XRPC client for calling your hatk endpoints.
---

The `hatk/xrpc-client` package provides a typed client for calling XRPC endpoints from the browser or any JavaScript environment.

## Setup

```typescript
// src/lib/api.ts
import { createClient } from 'hatk/xrpc-client'
import type { XrpcSchema } from '$hatk'
import { getAuthFetch } from './auth'

export const api = createClient<XrpcSchema>(
  typeof window !== 'undefined' ? window.location.origin : '',
  {
    fetch: (url, opts) => getAuthFetch()(url as string, opts),
  },
)
```

The `XrpcSchema` type is generated from your lexicons and provides full type safety for endpoint names, parameters, inputs, and outputs.

## Methods

### `api.query(nsid, params?)`

Call a query (GET) endpoint:

```typescript
const result = await api.query('xyz.statusphere.getStatuses', {
  limit: 30,
})
// result is typed: { statuses: StatusView[], cursor?: string }
```

Parameters are type-checked against the lexicon's parameter definitions.

### `api.call(nsid, input?, params?)`

Call a procedure (POST) endpoint with an optional request body:

```typescript
await api.call('dev.hatk.createRecord', {
  collection: 'xyz.statusphere.status',
  repo: viewerDid()!,
  record: { status: '🚀', createdAt: new Date().toISOString() },
})
```

The `input` argument is typed from the lexicon's input schema.

### `api.upload(nsid, data, contentType)`

Upload binary data (e.g., images):

```typescript
const result = await api.upload(
  'dev.hatk.uploadBlob',
  file,
  'image/jpeg',
)
// result: { blob: { ref: { $link: '...' }, ... } }
```

## Authentication

The client accepts a custom `fetch` function. Wire it to the OAuth client's authenticated fetch to automatically add DPoP headers:

```typescript
import { getAuthFetch } from './auth'

const api = createClient<XrpcSchema>(origin, {
  fetch: (url, opts) => getAuthFetch()(url as string, opts),
})
```

When the user is logged in, requests include `Authorization: DPoP <token>` and a `DPoP` proof header. When logged out, it falls back to plain `fetch`.

See [OAuth](/guides/oauth/) for setting up `getAuthFetch()`.

## Error handling

Failed requests throw an `Error` with the XRPC error message:

```typescript
try {
  await api.query('dev.hatk.unspecced.getPlay', { uri })
} catch (err) {
  // err.message: "NotFound" or "Missing required parameter: uri"
}
```

## Without SvelteKit

The client works in any JavaScript environment. Just pass a base URL:

```typescript
import { createClient } from 'hatk/xrpc-client'
import type { XrpcSchema } from './hatk.generated.ts'

const api = createClient<XrpcSchema>('https://my-appview.example.com')
```
